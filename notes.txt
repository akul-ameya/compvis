Phase 2: Synthetic Data Generation
2.1 Generator Setup

Model: Stable Diffusion v1.5 (runwayml/stable-diffusion-v1-5)
Scheduler: DPM-Solver++ (DPMSolverMultistepScheduler), 25 inference steps
Guidance scale: 7.5
Resolution: 512×512, then resize to 64×64 (Tiny ImageNet) or 32×32 (CIFAR-100) after generation
Mixed precision (float16) for speed on 4060
Batch size: 4 (safe for 8GB VRAM on 4060; test with 8 if memory allows)

2.2 Prompt Construction
For each class, derive the prompt from the WordNet label (Tiny ImageNet) or CIFAR-100 class name. Use four template variations per class:

"A photo of a {class_name}"
"A realistic photo of a {class_name}"
"An image depicting a {class_name}"
"A clear photograph of a {class_name}"

Cycle through templates across generated images for diversity.
For CIFAR-100: class names are already human-readable. Map directly.
For Tiny ImageNet: use the words.txt file mapping wnid to human-readable labels. Use the first listed synonym only.
2.3 Generation Budget
Generate up to 375 images per class for both datasets. This gives the pool from which allocation policies draw:

Tiny ImageNet: 200 classes × 375 = 75,000 images
CIFAR-100: 100 classes × 375 = 37,500 images

Generate once, cache to disk as PNG files organized as data/synthetic/{dataset}/{class_id}/{image_idx}.png. Never regenerate.
Estimate: ~75,000 images at ~4 images/batch, ~3 seconds/batch on 4060 = ~15.6 hours for Tiny ImageNet. ~7.8 hours for CIFAR-100. Run overnight or in parallel sessions.
2.4 Generation Quality Logging
During generation, save:

Total wall-clock time
Images per second throughput
Random seeds used per batch

After generation, compute and store:

Global FID (all real 5% subset vs all synthetic at each budget level) using torchmetrics or clean-fid. Do NOT compute per-class FID — statistically invalid with 25 real images per class.
Per-class synthetic quality proxy: for each class, extract penultimate-layer features from a frozen pretrained ResNet-18, compute mean cosine distance between each synthetic image and the real class centroid. Store as a CSV: class_id, mean_distance, std_distance, intra_synthetic_diversity. This serves as the fidelity proxy for allocation policies.


Phase 3: Model Training Infrastructure
3.1 Architectures
Two backbones. Both pretrained on ImageNet from torchvision:
ResNet-18:

torchvision.models.resnet18(weights=ResNet18_Weights.IMAGENET1K_V1)
Replace final fc layer: model.fc = nn.Linear(512, num_classes)

MobileNetV3-Small:

torchvision.models.mobilenet_v3_small(weights=MobileNet_V3_Small_Weights.IMAGENET1K_V1)
Replace classifier head: model.classifier[-1] = nn.Linear(1024, num_classes)
Note: use Small, not Large. Large is too heavy and offers no additional insight for this study.

3.2 Training Protocol
Identical across all pipelines and architectures:

Optimizer: AdamW, lr=3e-4, weight_decay=0.01
Scheduler: CosineAnnealingLR with T_max = max_epochs
Mixed precision training (torch.cuda.amp)
Early stopping: patience 7 on validation top-1 accuracy
Max epochs: 30
Batch size: 64
Loss: Cross-entropy (modified in Phase 5 for synthetic-aware weighting)

3.3 Pipeline Definitions
Four core pipelines, each defined by its training data composition:
Pipeline 1 — Baseline: Train on 5% real subset only (5,000 images for Tiny ImageNet, 2,500 for CIFAR-100). Standard augmentations. Standard cross-entropy loss.
Pipeline 2 — Uniform DiffusionBoost: Train on 5% real + uniform synthetic allocation. Run at three budgets: 5x (125 syn/class), 10x (250 syn/class), 15x (375 syn/class). Synthetic images are loaded from the cached pool. Combined dataset uses standard augmentations on all images (real and synthetic). Standard cross-entropy loss.
Pipeline 3 — Adaptive DiffusionBoost: Train on 5% real + adaptively allocated synthetic. Same total budget as Uniform 15x, but distributed per-class according to the allocation policy. Details in Phase 4.
Pipeline 4 — Ceiling: Train on 100% real data. Standard augmentations. Standard cross-entropy loss.
3.4 Training Loop Implementation
Each training run must:

Log epoch-level: train_loss, val_loss, val_top1_acc, val_top5_acc, learning_rate
Save best model checkpoint (by val_top1_acc)
Save final model checkpoint
Save training curves as JSON for later plotting
Track total training time (wall-clock)

Build a single train_pipeline(config, train_dataset, val_dataset, model) function that handles all pipelines. The only difference between pipelines is the dataset passed in.

Phase 4: Allocation Policies
4.1 Class-Level Diagnostics (extracted from Baseline model)
After training the Baseline model, extract per-class properties from the validation set. For each class c:

baseline_acc_c: per-class top-1 accuracy on validation
mean_confidence_c: mean softmax confidence on correct-class predictions
prediction_entropy_c: mean entropy of softmax distribution over validation samples
feature_compactness_c: mean L2 distance of each sample's penultimate feature to the class centroid
feature_separation_c: L2 distance between class centroid and nearest other class centroid
nearest_centroid_margin_c: difference between distance to own centroid and distance to nearest other centroid
synthetic_quality_c: mean cosine distance between synthetic images and real class centroid (from Phase 2.4)

Store as a DataFrame. This is the input to all allocation policies.
4.2 Allocation Policies
Given total budget B = 200 × 375 = 75,000 (Tiny ImageNet) or 100 × 375 = 37,500 (CIFAR-100) at 15x:
Policy 1 — Uniform: Each class gets B / num_classes images. This is Pipeline 2 at 15x.
Policy 2 — Hard-class: Budget proportional to (1 - baseline_acc_c). Normalize so total = B. Clamp to [min_floor, max_cap] where min_floor = 50, max_cap = 375.
Policy 3 — Uncertainty-based: Budget proportional to prediction_entropy_c. Same normalization and clamping.
Policy 4 — Predicted-utility: Train a ridge regression model predicting utility(c) = acc_c(Uniform_15x) - acc_c(Baseline) from the class-level features listed in 4.1. Use 10-fold cross-validation. Report R², feature importances. Allocate budget proportional to max(0, predicted_utility_c). Same normalization and clamping.
For each policy, output a CSV: class_id, allocated_count. Then construct the training dataset by sampling that many synthetic images per class from the cached pool.
4.3 Cross-Architecture Transfer Test
Compute allocation policy on ResNet-18 class-level features. Apply the same allocation to MobileNetV3 training. Report whether MobileNetV3 benefits from ResNet-18-derived allocation. This tests whether class-level utility is architecture-independent.

Phase 5: Synthetic-Aware Loss Reweighting (Architectural Contribution)
This is the novel methodological contribution required for the Methodology/Innovation rubric.
5.1 Concept
Standard cross-entropy treats all training samples equally. Synthetic samples that are far from the real distribution (high feature distance to real centroid) should be downweighted to avoid learning from noisy generations. Synthetic samples that are close to real should be upweighted as they provide genuine diversity.
5.2 Implementation
During training, for each batch:

Maintain a precomputed lookup: for every synthetic image in the training set, store its cosine similarity to the real class centroid (computed once before training using frozen pretrained ResNet-18 features). Real images get weight 1.0.
For synthetic images, compute weight as:

   w_i = sigmoid(lambda * (sim_i - mu_c))
where sim_i is the cosine similarity of synthetic image i to its class centroid, mu_c is the mean similarity across all synthetic images of class c, and lambda is a temperature hyperparameter (default: 5.0).

Apply as sample-level loss weighting:

   loss = (1/N) * sum(w_i * CE(f(x_i), y_i))

Real images always have w_i = 1.0.

5.3 Ablation Integration
Run every Adaptive DiffBoost experiment both WITH and WITHOUT this loss reweighting. The difference is one row in the ablation table. This isolates the contribution of the loss function from the allocation policy.

Phase 6: Evaluation Framework
After training each pipeline, evaluate the saved best checkpoint on the following axes. Build a single evaluate_model(model, val_loader, config) function that returns all metrics as a dictionary.
6.1 Accuracy

Top-1 accuracy (overall)
Macro accuracy (mean of per-class accuracies)
Worst-20-class accuracy (mean accuracy of the 20 lowest-performing classes)
Per-class accuracy array (200 or 100 values)

6.2 Calibration

Expected Calibration Error (ECE) with 15 bins
Compute reliability diagram data (bin_confidences, bin_accuracies, bin_counts)
Post-hoc temperature scaling: optimize temperature T on validation set using NLL, then recompute ECE. Report both raw ECE and calibrated ECE, and the optimal T.

6.3 Robustness
Apply three corruptions to each validation image and measure accuracy:

Gaussian noise: sigma=0.1
Gaussian blur: kernel_size=5, sigma=1.0
Brightness shift: +0.2 (clamp to [0,1])

Report clean accuracy and accuracy under each corruption. Compute mean corruption accuracy.
6.4 Per-Class Analysis

Per-class accuracy delta: acc_c(DiffBoost) - acc_c(Baseline) for each class
Classify each class as: beneficial (delta > 2pp), neutral (-2pp to +2pp), harmful (delta < -2pp)
Count: N_beneficial, N_neutral, N_harmful for each pipeline
Compare these counts across Uniform vs Adaptive allocation

6.5 Feature Coverage
Extract penultimate-layer features for all validation samples. Compute:

Within-class compactness: mean L2 distance to own class centroid, averaged over classes
Between-class separation: mean L2 distance between all pairs of class centroids
Nearest-centroid margin: per-class, distance to own centroid minus distance to nearest other centroid, averaged

6.6 Representation Geometry
Eigenvalue spectrum: Compute covariance matrix of penultimate features over validation set. Extract eigenvalues. Compute effective rank = exp(entropy of normalized eigenvalue distribution).
Linear probe: Freeze the trained backbone. Train a new linear classifier from scratch on the 5% real subset only (NOT on synthetic data). Use SGD, lr=0.01, 20 epochs. Report linear probe accuracy. This measures feature quality independent of the classifier head.
CKA similarity: Compute linear CKA between penultimate features of each pipeline's model on the same validation set. Report pairwise CKA matrix: Baseline-Uniform, Baseline-Adaptive, Baseline-Ceiling, Uniform-Ceiling, Adaptive-Ceiling. Use the standard formulation from Kornblith et al. 2019 (NOT the deconfounded version — standard CKA is appropriate here since all models are evaluated on the same single dataset).
